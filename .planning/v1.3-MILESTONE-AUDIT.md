- The highest-leverage product win was not new runtime capability; it was making every native page, docs entrypoint, and host-proof lane tell the same honest story.
- Keeping `/ops/jobs/oban` explicitly bounded and read-only reduced ambiguity without forcing a premature native generic queue or job dashboard rewrite.
- The example host remains the right proof surface, but it only stays trustworthy if its fixture migrations evolve with the runtime schema.

## Deferred v1.4+ Wedges

- `v1.4` should deepen operator forensics, retention history, and runbook-guided remediation rather than reopening generic dashboard scope.
- Richer automation or machine-facing surfaces remain deferred until the native control-plane vocabulary and support-truth boundaries have proved stable in real use.
- Any broader native replacement for generic Oban Web queue or job inspection remains explicitly deferred beyond v1.3.
