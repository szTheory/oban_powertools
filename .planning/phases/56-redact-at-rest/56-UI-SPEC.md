## Visual Hierarchy & Interaction Contract

Primary screen: `/ops/jobs` job detail view.

Focal point: the Args card is the first redaction-aware surface. When `meta["__redacted_fields__"]` is present, the operator must be able to identify missing arg fields without leaving the Args/Meta card cluster.

Visual priority:

| Priority | Element | Contract |
|----------|---------|----------|
| 1 | Args card redaction rows | Show each redacted arg field inline with the rendered args output as `field: Redacted at enqueue` so absence is intentional, not mistaken for missing data. |
| 2 | Meta card disclosure | Show `Fields redacted at enqueue: [:ssn, :token]` from `meta["__redacted_fields__"]` near the existing meta JSON display. |
| 3 | Existing job metadata and Recorded Output card | Keep current card order: job details, Args/Meta side by side, Recorded Output, Errors, Attempt History. Do not introduce a new top-level card for redaction. |

Interaction rules:

| Interaction | Contract |
|-------------|----------|
| Read-only disclosure | Phase 56 adds passive operator visibility only. No new destructive, retry, discard, edit, or reveal action is introduced. |
| Long field lists | Wrap redacted field lists onto multiple lines inside the existing card; never force horizontal page scroll outside the JSON `<pre>` regions. |
| DisplayPolicy fallback | If host display policy raises or returns invalid data, retain the existing fallback behavior and do not expose raw args as a recovery path. |
| Accessibility | Redaction disclosure is text, not color-only. The phrase `Redacted at enqueue` must appear in visible copy for every field listed by the default renderer. |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Open Job Detail |
| Empty state heading | No fields redacted at enqueue |
| Empty state body | This job has no `__redacted_fields__` metadata. Args are displayed according to the configured display policy. |
| Error state | Redaction metadata could not be rendered. Review the job meta payload or display policy configuration. |
| Destructive confirmation | none: no destructive actions in this phase |

Required visible strings:

| Surface | Copy |
|---------|------|
| Job detail redaction summary | `Fields redacted at enqueue: [:ssn, :token]` |
| Default args renderer placeholder | `Redacted at enqueue` |
| Fallback behavior | `[redacted]` remains the bounded fallback for invalid or raising display policy code |

Copy rules:

- Do not use `Hidden`, `Masked`, or `Encrypted` for fields dropped by `redact:`. The persisted field is absent, not encrypted or display-masked.
- Do not imply errors, meta, stacktraces, limiter partition keys, or recorded output are scrubbed unless that exact behavior is implemented by the backend plan.
- Use `redacted at enqueue` to distinguish persistence-time deletion from display-time `DisplayPolicy` masking.
