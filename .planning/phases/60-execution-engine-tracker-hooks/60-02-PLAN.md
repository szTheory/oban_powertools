  <behavior>
    - Test 1: Given a `job` without a `batch_id` in meta, `record_progress` returns `{:ok, :ignored}`.
    - Test 2: Given a valid `batch_id`, it idempotently inserts into `BatchJob` (returning `{:ok, :duplicate}` if called twice).
    - Test 3: On first insert, increments the `Batch` `success_count` (if `:success`) or `discard_count` (if `:discard`).
    - Test 4: When `success_count + discard_count == total_count` and `completed_at` is nil, sets `completed_at` and inserts a `Callback` (event: `"batch.completed"` if discard_count == 0, else `"batch.exhausted"`), returning `{:ok, :completed}`.
  </behavior>
  <action>
    Reference analog `lib/oban_powertools/cron.ex` from PATTERNS.md for module structure and conventions.
    Create `ObanPowertools.Batch.Tracker`. Define `record_progress(repo, %Oban.Job{}, state)` where state is `:success` or `:discard`.
    1. Extract `batch_id` from `job.meta["batch_id"]`. If nil, return `{:ok, :ignored}`.
    2. Execute `repo.insert_all` on `ObanPowertools.BatchJob` with `[batch_id: batch_id, job_id: job.id, state: to_string(state), inserted_at: now, updated_at: now]` and `on_conflict: :nothing`.
    3. If `count == 1` from `insert_all`:
       a. Execute `repo.update_all` on `ObanPowertools.Batch` using `where: b.id == ^batch_id` and `update: [inc: [{^inc_field, 1}]]`, using `select: [:id, :total_count, :success_count, :discard_count]`.
       b. If the returned batch's `success_count + discard_count == total_count`, execute an `Ecto.Multi` transaction:
          - `Multi.update_all` on `Batch` `where: b.id == ^batch_id and is_nil(b.completed_at)`, updating `completed_at` to `now` and `status` to `"completed"` or `"exhausted"`.
          - If the previous `update_all` updated 1 row, `repo.insert` an `ObanPowertools.Callback` with `batch_id: batch_id`, `event: "batch.completed" | "batch.exhausted"`, `status: "pending"`, and `dedupe_key: "#{event}-#{batch_id}"`.
  </action>

  </verify>
