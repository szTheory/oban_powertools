<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| preview metadata -> execute audit | Continuity facts pass from ephemeral preview state into immutable audit evidence. |
| audit metadata -> forensic projection | Structured metadata is transformed into operator-visible chronology and runbook context. |
| runbook continuity -> operator action | Continuity wording influences remediation confidence and escalation decisions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-01-01 | Tampering | `Lifeline` continuity metadata | mitigate | Build runbook context from server-side facts only (`plan_hash`, selectors, action, target) and never from client text params. |
| T-35-01-02 | Repudiation | Attempt-state evidence | mitigate | Persist explicit attempt states (`previewed`, `succeeded`, `drifted`, `expired`, `consumed`) in preview/audit metadata and assert in tests. |
| T-35-01-03 | Information Disclosure | URL and metadata boundaries | mitigate | Keep reason text, rendered copy, and preview internals out of URL selectors; use stable forensic selectors only. |
| T-35-01-04 | Spoofing | Ownership/venue rendering | mitigate | Continue rendering ownership via normalized triad labels, not freeform metadata strings. |
| T-35-01-05 | Denial of Service | Forensics projection fallback | mitigate | When continuity metadata is missing, degrade to existing runbook behavior without crashes or missing page render. |
</threat_model>
