<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host worker callback -> Powertools wrapper | Host-defined hook code runs inside the Oban job process and can raise, throw, exit, log, or perform side effects. |
| Powertools wrapper -> Oban executor | `perform/1` return or raised/caught process failure determines Oban retry, discard, cancel, or success handling. |
| Powertools hook dispatcher -> public telemetry | Hook health signals cross into the public low-cardinality metrics API. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-01 | Denial of Service / Tampering | `ObanPowertools.Worker.Hooks.safe_invoke/4` | mitigate | Catch hook exceptions, throws, and exits; warning-log them; emit outcome `"crash_caught"`; never let hook failure alter the `perform/1` return or preserved process exception. Verify with hook-crash tests for start, success, failure, and discard paths. |
| T-53-02 | Tampering | `ObanPowertools.Worker.Hooks.after_result/3` and `after_exception/5` | mitigate | Classify final attempts with `job.attempt >= job.max_attempts`, route final-attempt failures to `on_discard/2` only, and add the D-32 non-double-fire test. |
| T-53-03 | Information Disclosure | `ObanPowertools.Telemetry.execute_worker_hook_event/3` | mitigate | Emit metadata keys exactly `:hook` and `:outcome` with string values only; telemetry tests assert exact key set and metric tags stay within `contract/0`. |
| T-53-04 | Repudiation | Worker hook side effects | mitigate | Runtime treats hook return values as ignored and non-durable; Plan 53-02 documents that hooks are observe-only, in-process, outside transactions, and not retried independently. |
| T-53-05 | Denial of Service | Synchronous hook execution in job process | accept | D-19 requires synchronous in-process hooks and forbids async registry/supervision. The phase documents this support truth; this plan avoids adding hidden async machinery. |
| T-53-SC | Tampering | npm/pip/cargo installs | mitigate | No package-manager install tasks exist in this plan; Phase 53 research package audit says no new packages. |
</threat_model>
