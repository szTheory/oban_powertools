<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub `workflow_run` event → automerge runner | `head_branch`/`head_sha` from a prior CI run are attacker-influenceable inputs that decide whether a PR is merged |
| Release PR contents → squash-merge into `main` | An untrusted PR body/title crosses into a privileged merge + dispatch action |
| New workflow YAML on any PR → CI runners | Malformed or malicious workflow YAML could change CI behavior if not statically checked |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-52-01 | Tampering | `release-pr-automerge.yml` stale `workflow_run` SHA | mitigate | `head_oid != HEAD_SHA` guard (line 119) exits 0 on stale events — present in committed workflow; confirmed by Task 2 item 4 |
| T-52-02 | Tampering | Untrusted PR title used in bash merge decision | mitigate | Title validated against `^chore\(main\):\ release\ ` regex (line 124) before any `gh pr merge`; mismatch exits without merging — present in committed workflow |
| T-52-03 | Elevation of Privilege | `--admin` squash-merge fallback | accept | `--admin` is fallback-only, reached only after branch-protection catch-up timing (12×30s loop); release automation requires it. Permissions scoped to `contents/pull-requests/actions: write` only — no `id-token`/`packages` |
| T-52-04 | Tampering | Malformed/malicious workflow YAML reaching `main` | mitigate | actionlint lane (Task 1) statically validates all `.github/workflows/*.yml` on every PR and is enforced by `ci-gate` |
| T-52-SC | Tampering | GitHub Action supply chain (`raven-actions/actionlint`) | mitigate | Action SHA-pinned to `205b530c5d9fa8f44ae9ed59f341a0db994aa6f8 # v2.1.2` (verified via GitHub API in RESEARCH); consistent with the all-actions-SHA-pinned convention. Not an npm/pip/cargo install — no slopcheck applies |
</threat_model>
