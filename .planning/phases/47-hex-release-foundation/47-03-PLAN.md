## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions CI → hex.pm | The publish job authenticates to hex.pm with HEX_API_KEY; this secret is the supply-chain trust anchor. |
| working tree → published tarball | `mix hex.publish` ships the filesystem at run time, not the last commit (clean-tree gate mitigates). |
| release-please-action → repo (PRs, tags) | The action writes to the repo via GITHUB_TOKEN with scoped permissions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-47-07 | Information Disclosure | HEX_API_KEY secret | mitigate | Stored as a GitHub Actions secret (masked in logs), publish-scoped not full-account (D-04); never echoed in `run:` steps; referenced only via `env:`. |
| T-47-08 | Tampering | publishing a dirty working tree | mitigate | `git status --porcelain` clean-tree gate as the first publish step fails the job on any uncommitted file, preventing `.planning/`, local secrets, or uncommitted code leaking into the tarball (D-00e, Pitfall 17). |
| T-47-09 | Elevation of Privilege | workflow GITHUB_TOKEN permissions | mitigate | Top-level `permissions:` grants only `contents: write` + `pull-requests: write` (least privilege); no broader scopes. |
| T-47-10 | Spoofing | wrong first-release version | mitigate | Manifest seeded at `0.0.0` + `Release-As: 0.5.0` footer ensures the published version is the intended 0.5.0; operator verifies the Release PR proposes 0.5.0 before merge (T-47 Task 5). |
| T-47-SC | Tampering | release-please-action supply chain | mitigate | Pinned to a specific major from the canonical `googleapis` org (Task 1 confirms current stable); no third-party/forked actions. No npm/pip/cargo installs in this phase (no slopcheck applicable per RESEARCH Package Legitimacy Audit). |
</threat_model>
