<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Workflow terminal commit -> callback delivery | Delivery must remain a separate durable obligation so callback failure cannot falsify workflow truth. |
| Multi-node dispatcher -> callback outbox rows | Without explicit row leasing, concurrent nodes can race and double-deliver the same callback. |
| Runtime schema -> installer and fixture migrations | If outbox delivery fields exist only in runtime code, supported hosts and test fixtures drift from the real contract. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | T | callback delivery ownership | mitigate | Add explicit lease-safe callback claiming instead of scan-and-send delivery. |
| T-18-02 | R | callback failure evidence | mitigate | Persist bounded failure state, retry schedule, and attempt increments on every failed delivery. |
| T-18-03 | D | supported-host schema parity | mitigate | Land callback-outbox schema changes together in runtime, installer, example hosts, and test-support migrations. |
</threat_model>
