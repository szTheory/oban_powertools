<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Step mirror -> await truth | If `workflow_steps` owns wait detail instead of mirroring it, diagnosis and correctness can drift. |
| External correlation -> workflow-scoped signal authority | Correlation-only matching is untrusted until durable workflow authority is resolved and stored. |
| Runtime schema -> installer and supported hosts | If migrations drift from runtime fields, upgrade safety and support truth break immediately. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | T | step wait mirror | mitigate | Store detailed wait truth only on `workflow_awaits` and keep step projection thin with an `active_await_id` pointer. |
| T-19-02 | S | correlation-only signal targeting | mitigate | Require durable workflow authority on signal rows before a wait can be consumed. |
| T-19-03 | D | schema parity across supported hosts | mitigate | Change runtime schemas and installer/example/test migrations together and verify the same fields/indexes exist everywhere. |
</threat_model>
