<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| forensic bundle -> runbook entry | Durable evidence is transformed into recommended operator guidance. |
| runbook entry -> operator decision | Advisory copy can be mistaken for executed remediation or owned escalation. |
| URL selectors -> bundle reconstruction | Untrusted params choose a resource scope for runbook and evidence reconstruction. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-02-01 | Spoofing | Runbook ownership labels | mitigate | Normalize ownership to exactly `Powertools-native`, `Oban Web bridge`, and `host-owned follow-up`; render label before path/action intent. |
| T-34-02-02 | Tampering | `Forensics.selectors/1` consumers | mitigate | Reuse existing selector allowlist only; do not accept rendered copy, preview tokens, or runbook text from params. |
| T-34-02-03 | Repudiation | Runbook advisory status | mitigate | Include unsupported boundary copy that Phase 34 does not execute, persist sessions, or record remediation attempts. |
| T-34-02-04 | Information Disclosure | Evidence links | mitigate | Evidence links use stable resource selectors and never include reason prose, preview tokens, or copied diagnosis text. |
| T-34-02-05 | Elevation of Privilege | Forensics LiveView | mitigate | Add no mutation events, confirmation forms, or native action execution; page remains read-only through `LiveAuth.authorize_page/3`. |
| T-34-02-06 | Denial of Service | Runbook rendering | accept | Entry data is bounded by existing bundle lists and no new unbounded queries are introduced. |
</threat_model>
