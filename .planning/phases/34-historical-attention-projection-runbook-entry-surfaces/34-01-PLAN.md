<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| retained history -> overview | Historical facts influence what operators inspect next; projection must not overstate certainty or imply raw-feed completeness. |
| overview link -> LiveView route | Links cross from rendered HTML into native and forensic destinations using URL selectors. |
| read-only page -> action intent | Overview recommendations can look action-oriented even though the page is read-only and advisory. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-01-01 | Tampering | `AttentionProjection.project_bucket/2` | mitigate | Accept only read-model candidate data, normalize completeness, sort deterministically, and cap at three exemplars so injected or excessive history cannot become a feed. |
| T-34-01-02 | Information Disclosure | `EngineOverviewLive` links | mitigate | Render only stable selector URLs; tests must reject `attention_reason`, `reason`, `copy`, and `preview_token` query params. |
| T-34-01-03 | Repudiation | Overview attention reasons | mitigate | Each attention reason must include `evidence_path` or explicit degraded completeness so the support truth can be inspected in forensics. |
| T-34-01-04 | Elevation of Privilege | Overview CTAs | mitigate | Keep overview CTAs navigational only; do not add mutation events, preview tokens, or remediation execution controls. |
| T-34-01-05 | Spoofing | Ownership labels | mitigate | Render `Powertools-native`, `Oban Web bridge`, or `host-owned follow-up` from structured exemplar metadata at the decision point. |
</threat_model>
