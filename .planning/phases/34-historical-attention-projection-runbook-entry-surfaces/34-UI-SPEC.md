## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Open runbook entry |
| Empty state heading | No historical attention needed |
| Empty state body | Current state and retained history do not identify a safe runbook path right now. Review the existing diagnosis surfaces or open forensics for evidence detail. |
| Error state | Runbook guidance is unavailable because the evidence bundle could not be assembled. Refresh the page, then open the forensic timeline for the same resource. |
| Destructive confirmation | None in Phase 34. Runbook entries are advisory and must not execute, persist, or confirm remediation. |

Runbook and refusal-adjacent wording must follow this order: `outcome -> concise reason -> legal next move -> venue -> evidence`. Ownership/venue labels must use exactly: `Powertools-native`, `Oban Web bridge`, and `host-owned follow-up`.

## Phase Interaction Contract

Historical attention projection must extend existing overview buckets, not add a new feed, dashboard band, inbox, or raw history table. Each affected bucket may show one to three historical exemplars, ordered by diagnosis impact rather than recency alone. Every exemplar must include a label, a concise fact/reason, evidence completeness when partial or unavailable, a venue/ownership label at the decision point, and one honest next path.

Compact runbook entry summaries may appear on the overview and selected native drilldowns only when Powertools can state a safe next investigative or remediation path. The compact shape is: title, why it matters now, top prerequisite or caution, ordered next path label, venue/ownership label, and evidence link. If no safe path exists, render guidance text or refusal-adjacent next steps; do not show disabled mystery controls.

The canonical deep runbook entry belongs on `/ops/jobs/forensics` near the diagnosis summary and before the timeline. It must render: diagnosis state, why it matters now, prerequisites, cautions, recommended order of operations, ownership/venue for each follow-up path, evidence link, unsupported boundary copy, and completeness state.

Stable navigation continuity must use only durable selectors: `resource_type`, `resource_id`, `workflow_id`, `step`, `incident_fingerprint`, and `view`. Rendered diagnosis copy, attention reason text, preview tokens, refusal prose, and runbook wording must stay off URLs.

## Surface Contracts

| Surface | Required Phase 34 Behavior |
|---------|----------------------------|
| `/ops/jobs` overview | Preserve existing diagnosis-first bucket grid. Historical attention may change exemplar order, exemplar reason, and next path only when it changes the next safe operator action. Cap exemplars at three per bucket. |
| Limiter drilldown | Keep the existing history summary card. Add compact runbook guidance only for blocked, recently restored, cooling down, policy-caused, or partial-evidence states with honest next paths. |
| Cron drilldown | Keep the existing history summary card. Add compact runbook guidance only for missed-fire, delayed-fire, overlap-relevant, manual-run, unknown-window, or partial-evidence states with honest next paths. |
| Workflow drilldown | Add runbook guidance only around blocked steps or refused mutation states, reusing shared refusal vocabulary and showing venue before action. |
| Lifeline drilldown | Add runbook guidance around active incidents and repair-preparation paths without claiming Phase 35 remediation continuity. |
| `/ops/jobs/forensics` | Own the canonical deep runbook entry and keep the reading order diagnosis summary, runbook entry, timeline, related evidence, linked resources, legal next paths, evidence completeness. |

## State Contract

| State | Visual Treatment | Required Copy Behavior |
|-------|------------------|------------------------|
| Complete evidence | White panel with slate nested evidence rows | State the supported next path directly and link to evidence. |
| Partial evidence | Amber or slate boundary note inside the runbook card | Use `partial evidence`; explain what is known and what cannot be inferred. |
| History unavailable | Amber boundary note, no primary CTA unless a safe diagnostic path exists | Use `history unavailable`; route to current diagnosis or forensics, not remediation. |
| Unknown | Slate nested row with explicit unknown label | Use `unknown`; avoid causal certainty and avoid action-intent verbs. |
| Bridge-only | Border or plain link treatment, not a filled primary button | Use `Oban Web bridge` and `Inspection only` at the decision point. |
| Host-owned follow-up | Border/plain guidance treatment, not a filled native button | Use `host-owned follow-up`; say Powertools does not own delivery or external runbook truth. |
| Unsupported/premature | Amber refusal-adjacent panel | Explain the reason and the next legal venue; do not render a disabled unlabeled action. |
