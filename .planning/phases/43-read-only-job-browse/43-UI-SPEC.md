## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `px-2 py-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `mt-2`, `space-y-2`) |
| md | 16px | Default element spacing (`p-4`, `px-4 py-3`, `gap-4`) |
| lg | 24px | Section padding (`p-6`, `space-y-6`, `gap-6`) |
| xl | 32px | Layout gaps (`space-y-8`) |
| 2xl | 48px | Major section breaks (not currently used; reserved) |
| 3xl | 64px | Page-level spacing (not currently used; reserved) |

Source: extracted from existing LiveViews — `p-6` outer page padding, `px-4 py-3` table cells, `gap-4` / `gap-6` grid gaps match 8-point scale exactly.

Exceptions:
- Table cell padding: `px-4 py-3` (16px / 12px) — established pattern for data tables; do not change for consistency
- Badge padding: `px-2 py-1` (8px / 4px) — established compact badge pattern; do not change

|------|------|--------|-------------|

- `h1` always `text-2xl font-semibold`
- section headings always `text-base font-semibold`

## Component Inventory

These are the concrete UI building blocks for Phase 43. No new component abstractions are required — all patterns already exist in the codebase.

### Job List Page (`/ops/jobs/jobs?state=executing&...`)

1. **Page header** — `h1.text-2xl.font-semibold` "Jobs" + `p.text-sm.text-zinc-600` subtitle with native banner
2. **Read-only banner** — `p.rounded-lg.border.border-amber-200.bg-amber-50.px-4.py-3.text-sm.text-amber-800` (conditional on `@read_only?`) — from `LiveAuth.page_read_only_banner(:jobs)`
3. **State tab bar** — horizontal row of `button` elements (one per Oban state, 7 total), using `view_toggle_class/1` pattern:
   - Active: `rounded border border-indigo-300 bg-indigo-50 px-3 py-2 text-sm font-medium text-indigo-700`
   - Inactive: `rounded border border-slate-200 bg-white px-3 py-2 text-sm font-medium text-slate-600`
   - Each tab shows label + count in parens: `executing (12)`
   - State is always selected (no "all" tab); default state: `available` on mount if no URL param
4. **Filter bar** — `form` with three inline selects: Queue, Worker, Tags — `phx-change` triggers `push_patch` to update URL params
5. **Job table** — `div.overflow-hidden.rounded-lg.border.bg-white` wrapping `table.min-w-full.divide-y`:
   - `thead.bg-slate-50.text-left.text-sm` with `th.px-4.py-3.font-medium` columns: State, Worker, Queue, Job ID, Scheduled At, Attempts
   - `tbody.divide-y.text-sm` with `tr` rows, each `td.px-4.py-3`
   - Clickable row: entire row is navigable to `/jobs/:id` via `phx-click` + `JS.navigate`
   - State column: state badge using the state badge color table above
   - Worker column: short module name (last segment only, e.g., `IngestWorker`)
6. **Empty state** — `div.rounded-lg.border.bg-white.p-6` with `h2.text-base.font-semibold` + `p.mt-2.text-sm.text-zinc-600`
7. **Pagination controls** — Previous / Next buttons using `rounded border px-3 py-2 text-sm` (ghost button), disabled state `cursor-not-allowed text-zinc-400`

### Job Detail Page (`/ops/jobs/jobs/:id`)

1. **Page header** — `h1.text-2xl.font-semibold` "Job #{id}" + back link `text-indigo-700 underline` to return to list (preserving URL filter state)
2. **Read-only banner** — same pattern, from `LiveAuth.page_read_only_banner(:job_detail)`
3. **Identity card** — `div.rounded-lg.border.bg-white.p-4`:
   - Worker, Queue, State badge, Job ID, Attempt count
   - Timing fields: `inserted_at`, `scheduled_at`, `attempted_at`, `completed_at` / `cancelled_at` / `discarded_at` (show only non-nil values)
4. **Args panel** — `div.rounded-lg.border.bg-white.p-4`:
   - `h2.text-base.font-semibold` "Args"
   - Content: DisplayPolicy-redacted output — either raw JSON in `pre.text-sm.bg-slate-50.p-3.rounded.overflow-x-auto` or host-provided string/map
5. **Meta panel** — same pattern as Args panel, heading "Meta"
6. **Errors panel** — `div.rounded-lg.border.bg-white.p-4`:
   - `h2.text-base.font-semibold` "Errors"
   - Empty: `p.text-sm.text-zinc-600` "No errors recorded."
   - Each error record: `div.rounded.border.bg-slate-50.p-3.text-sm` with attempt number, error text, backtrace (collapsed by default if long)
7. **Attempt history panel** — `div.rounded-lg.border.bg-white.p-4`:
   - `h2.text-base.font-semibold` "Attempt History"
   - Table or list of attempt entries: attempt number, state at attempt, timing

Layout: `div.space-y-6.p-6` outer wrapper. Detail panels stack vertically on mobile; two-column grid on xl for args/meta side by side: `xl:grid-cols-2 gap-6`.

No mutation controls appear anywhere in Phase 43 — the detail page is strictly read-only. Phase 44 attaches action controls to this same route.

## Interaction Contract

### Filter Flow
1. User arrives at `/ops/jobs/jobs` — no state param → mount selects `available` as default, pushes patch to `/ops/jobs/jobs?state=available`
2. User clicks state tab → `phx-click="set_state"` with `phx-value-state` → `push_patch` updates URL
3. User changes queue/worker/tags select → `phx-change="filter"` → `push_patch` updates URL params
4. `handle_params/3` always re-runs the query from URL params — browser back/forward preserved automatically
5. All filter changes are additive-AND: state AND queue AND worker AND tags

### Row Navigation
- Entire `tr` row is clickable: `phx-click="select_job"` + `phx-value-id` → `JS.navigate("/ops/jobs/jobs/#{id}")`
- No side panels — full-page navigation per CONTEXT.md D-08
- Back link on detail page navigates to `/ops/jobs/jobs` preserving the last-known filter params (pass through assigns or URL param)

### Tab Active State
- Active tab receives `view_toggle_class(true)` → indigo treatment
- Only one tab active at a time
- Tab counts update on mount and on filter change (count query per visible tab set)

### Unauthorized State
- `LiveAuth.authorize_page/3` failure → `redirect(socket, to: "/")` (existing pattern)
- `@read_only?` flag → read-only banner renders above content; job data still visible; no mutation surface to disable in Phase 43

### DisplayPolicy Redaction
- `render_job_field(:job_args, value, context)`:
  - `nil` return → render raw JSON in `<pre>` block
  - `String` return → render string as-is in `<pre>` block
  - `Map` return → render field-level redacted map as formatted JSON in `<pre>` block
- Same logic for `:job_meta`
- Never crash on redaction failure — fall back to `[redacted]` with `text-zinc-500`
