<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer stream to Oban job storage | Caller-provided stream items become Oban jobs in `oban_jobs`. |
| Developer options to Powertools batch rows | `total_count`, `chunk_size`, `batch_id`, and `name` control persistence behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-61-01 | Denial of Service | `Batch.insert_stream/2` | mitigate | Validate `total_count` and `chunk_size`; default to `chunk_size: 1_000`; insert with one `Oban.insert_all/2` call per chunk; never wrap the whole stream in one transaction. |
| T-61-02 | Tampering | deterministic `batch_id` | mitigate | Treat existing caller-supplied `batch_id` as `:batch_id_exists`; never append to an existing batch. |
| T-61-03 | Information Disclosure | batch metadata | mitigate | Inject only `"batch_id"` and optional `"batch_name"` into job meta; never copy business stream payloads into batch metadata. |
| T-61-04 | Repudiation | partial insert failure | mitigate | Persist `"insert_failed"` status, `inserted_count`, `insert_failed_chunk`, `insert_failure`, and `insert_failed_at` before returning `InsertError`. |
| T-61-SC | Tampering | Package installs | mitigate | No package-manager install tasks exist in this plan; RESEARCH.md Package Legitimacy Audit lists no packages. |
</threat_model>
