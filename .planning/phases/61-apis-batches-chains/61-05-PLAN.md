<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Downstream worker to JobRecord storage | Worker code reads predecessor output through job metadata references. |
| Persisted builder reference to BEAM execution | Callback dispatcher executes a module/function reference after restart. |
| Upstream output to downstream args | Upstream payload may influence downstream job args only through explicit builder code. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-61-01 | Denial of Service | output-dependent progression | mitigate | Output lookup failures do not spin or block; they mark callback rows failed with retry metadata and explicit errors. |
| T-61-02 | Elevation of Privilege | persisted args builders | mitigate | Require `ObanPowertools.Chain.ArgsBuilder` marker and function arity 2; reject anonymous functions, missing modules, missing marker, and missing function before persistence/execution. |
| T-61-03 | Information Disclosure | upstream payload handoff | mitigate | Store and pass upstream job ids; fetch payload from `JobRecord`; never auto-copy upstream payload into downstream args or callback payload. |
| T-61-04 | Repudiation | missing/expired/oversized/unrecorded output | mitigate | Return exact errors `:missing_upstream_job_id`, `:output_unavailable`, and `:output_expired`; progression persists failures in callback `last_error`. |
| T-61-SC | Tampering | Package installs | mitigate | No package-manager install tasks exist in this plan; RESEARCH.md Package Legitimacy Audit lists no packages. |
</threat_model>
