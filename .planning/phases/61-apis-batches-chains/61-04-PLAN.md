<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Worker result hook to callback outbox | Successful jobs with chain metadata create internal callback rows. |
| Callback row payload to next Oban job | Persisted metadata drives downstream job insertion after a BEAM restart. |
| Workflow callback dispatcher to host handler | Host workflow callbacks must not receive internal chain events. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-61-01 | Denial of Service | callback dispatcher claims | mitigate | Use `limit`, `lease_seconds`, availability checks, and `FOR UPDATE SKIP LOCKED` for chain claims; do not run massive chain work inside worker hooks. |
| T-61-02 | Tampering / Elevation of Privilege | chain callback payload | mitigate | Consume only normalized next-step descriptors produced by `Chain.insert/3`; Plan 61-05 validates builder behavior before executing dynamic args. |
| T-61-03 | Information Disclosure | chain callback payload | mitigate | Payload contains ids and next-step descriptors only; upstream output payloads are not copied into callback rows. |
| T-61-04 | Repudiation | failed progression | mitigate | Mark failed callbacks with attempts, retry availability, and `last_error` so Phase 62 can explain blocked chain progression. |
| T-61-SC | Tampering | Package installs | mitigate | No package-manager install tasks exist in this plan; RESEARCH.md Package Legitimacy Audit lists no packages. |
</threat_model>
