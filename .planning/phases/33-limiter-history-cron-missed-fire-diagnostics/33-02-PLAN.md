# 33-02 Plan: Cron Missed-Fire Diagnostics

## Goal

Add cron missed-fire, delayed-fire, and overlap-relevant history views that explain why scheduled work did not run when expected.

## Implementation

- Keep `oban_powertools_cron_slots` as the primary durable cron anchor.
- Add a scheduler coverage store so missing slots can be distinguished from unknown windows.
- Classify recent cron windows as on-time, missed fire, delayed claim, overlap-relevant, manual run, partial evidence, or unknown.
- Persist manual/source and overlap metadata needed to avoid guessing after the fact.
- Route `resource_type=cron_entry&resource_id=<name>` through the shared forensic bundle contract.

## Acceptance Criteria

- Healthy coverage plus a missing slot renders as a missed fire.
- Coverage gaps render as unknown or partial evidence, not false certainty.
- Manual runs are visible but do not rewrite schedule truth.
- Cron detail pages show a bounded history summary and forensic deep link.
