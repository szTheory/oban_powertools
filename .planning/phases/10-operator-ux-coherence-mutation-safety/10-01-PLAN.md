<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Native preview intent -> durable mutation write | Execute must consume a durable preview token and reject drifted, expired, or consumed previews before writes. |
| Low-risk cron action UX -> high-trust repair UX | Cron should converge on the same durability model without forcing lifeline-level friction onto every low-risk action. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Repudiation | preview lifecycle | mitigate | Persist preview state durably and attach preview token plus reason/audit consequence to native mutations. |
| T-10-02 | Tampering | execute path | mitigate | Re-check preview state server-side before execute and reject drifted/expired/consumed previews. |
| T-10-03 | Elevation of Privilege / support-truth drift | cron mutation flow | mitigate | Keep authorize-before-preview and authorize-before-execute ordering while moving cron onto the same durable preview model as lifeline. |
</threat_model>
