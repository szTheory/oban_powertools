<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| overview CTA metadata -> native destination selection | Only durable, destination-owned selectors may cross LiveView boundaries. |
| URL-owned focus state -> preview or mutation state | Query params may restore context but must not recreate preview tokens, draft reasons, or execute-ready state. |
| native control plane -> bridge inspection venue | Bridge handoffs must remain explicit and filtered instead of implying unsupported native parity. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-03 | T | selection continuity | mitigate | Standardize router-owned params and patch flows for every exact-focus native destination. |
| T-28-04 | I | cross-surface handoff honesty | mitigate | Limit exact deep links to destination-owned native read models and keep bridge destinations explicitly inspection-only. |
| T-28-05 | R | preview-state leakage | mitigate | Prohibit preview tokens and reason text from every Phase 28 URL contract and test for absence. |
</threat_model>
