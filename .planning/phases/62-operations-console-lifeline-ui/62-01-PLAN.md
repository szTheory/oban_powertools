<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser to LiveView params/events | Query params and event payloads can be forged and must be allowlisted or parsed before use. |
| LiveView to read model | UI must not own cross-table SQL or bypass redaction/copy-ready evidence contracts. |
| LiveView to Lifeline | UI must not mutate jobs or callbacks directly; Lifeline must re-fetch, authorize, preview, drift-check, execute, and audit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-62-01 | Tampering | `/ops/jobs/batches` query params | mitigate | Wave 0 tests assert URL-backed filters use allowlisted statuses and selector helpers before implementation. |
| T-62-02 | Information Disclosure | failed member and callback payload rendering | mitigate | Wave 0 read-model/LiveView tests require DisplayPolicy seams and copy-ready evidence rather than raw payload leakage. |
| T-62-03 | Elevation of Privilege | failed member retry events | mitigate | Wave 0 LiveView tests require read-only disabled controls and Lifeline preview/execute gates before mutation. |
| T-62-04 | Tampering | callback retry target state | mitigate | Wave 0 Lifeline tests require eligibility checks, drift detection, reason gating, single-use previews, and audit evidence. |
| T-62-SC | Tampering | package installs | accept | No package-manager install tasks exist in Phase 62; Package Legitimacy Audit says no external packages are installed. |
</threat_model>
