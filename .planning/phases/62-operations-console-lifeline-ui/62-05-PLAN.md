<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser events to LiveView | Status, filter, selected ids, callback ids, and reason text are client-controlled. |
| LiveView to read model | All batch data must come from `ObanPowertools.Batches`, not LiveView SQL. |
| LiveView to Lifeline mutation | Job and callback recovery must go through Lifeline preview/execute, not direct row updates. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-62-01 | Tampering | status/filter/page events | mitigate | Closed status allowlist, integer parsing, selector-based URL patches, and tests for selection reset. |
| T-62-02 | Information Disclosure | detail panels | mitigate | Render values supplied through Batches/DisplayPolicy seams; do not raw-render payloads from events or SQL in LiveView. |
| T-62-03 | Elevation of Privilege | failed job bulk retry | mitigate | Validate ids against server-derived retry-eligible set, authorize execute, and call Lifeline per job. |
| T-62-04 | Tampering | callback retry | mitigate | Validate callback id against current eligible callback rows, authorize preview/execute, and rely on Lifeline drift checks. |
| T-62-07 | Repudiation | partial bulk retry results | mitigate | Flash success and skipped/failed counts; Lifeline writes audit rows for executed repairs. |
| T-62-SC | Tampering | package installs | accept | No package-manager install tasks exist. |
</threat_model>
