<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| actor to Lifeline preview/execute | Actor permission controls whether preview or execute can proceed. |
| client-supplied target id to callback row | Target id is untrusted and must be re-fetched and eligibility-checked. |
| preview token to mutation | Preview token must be ready, unexpired, unconsumed, and non-drifted before mutation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-62-04 | Tampering | callback retry target | mitigate | Re-fetch callback on preview/execute, restrict eligibility to failed or expired-lease claimed rows, and recompute drift hash. |
| T-62-05 | Elevation of Privilege | Lifeline callback execution | mitigate | Reuse `authorize(actor, :preview_repair, attrs)` and `authorize(actor, :execute_repair, ...)` before mutation. |
| T-62-07 | Repudiation | callback retry audit | mitigate | Existing `apply_repair/6` writes `lifeline.repair_executed` with resource type `callback`, reason, preview token, and result. |
| T-62-08 | Denial of Service | callback retry target type | mitigate | Extend closed `TargetType.to_atom/1` with a bounded `"callback"` clause and no catch-all atom conversion. |
| T-62-SC | Tampering | package installs | accept | No package-manager install tasks exist. |
</threat_model>
