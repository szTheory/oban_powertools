## Phase 65 Layout & Interaction Contract

### Cross-Page "Select All" Banner (QRY-08)

When an operator toggles the select-all checkbox in the `JobsLive` table header:
1. It selects all jobs on the **current page**.
2. **Global Selection Trigger:** If the total number of jobs matching the current filter (`total_count`) is greater than the number of jobs visible on the current page, an inline banner appears bridging the table header and the first data row (or at the top of the table panel).
3. **Banner Styling:** Use `bg-indigo-50 border-y border-indigo-200 py-2 px-4 text-center text-sm`.
4. **Banner Content (Local Mode):** `All {N} jobs on this page are selected. <button>Select all {Total} jobs matching this filter</button>`. The button is a primary accent link (`text-indigo-600 font-semibold hover:underline`).
5. **Global Selection Mode:** When the operator clicks the "Select all {Total}..." button, the internal state shifts to `global_select: true`.
6. **Banner Content (Global Mode):** The banner text updates to `All {Total} jobs matching this filter are selected. <button>Clear selection</button>`.
7. **Action Scope:** Any subsequent bulk actions (e.g. Retry, Cancel, Discard) must recognize the `global_select` flag. The backend execution will need to stream or batch process jobs based on the filter criteria instead of an explicit ID list. The UI preview modal must accurately reflect the total count rather than the local page count.

### Deep Links (QRY-07)

- **Target:** Native job detail page (`/ops/jobs/:id`).
- **Surfaces:** All Lifeline repair previews/audit tables (`LifelineLive`) and evidence logs (`AuditLive`).
- **Implementation:** Whenever a job ID or job reference is displayed in these contexts, wrap it in a LiveView link using standard Phoenix routing: `<.link navigate={Selectors.jobs_path(@filter, id: job.id)} class="text-indigo-600 hover:underline">`.
- This ensures operators can fluidly navigate from audit trails and blockage views directly into single-job inspection without losing context or resorting to the Oban Web bridge unless strictly necessary.

### Accessibility & UX Polish
- The cross-page selection banner must be contained within an `aria-live="polite"` region so screen readers immediately announce "All {N} jobs on this page are selected".
- Ensure that the "Select all" button is a semantic `<button type="button">`.
- When in `global_select` mode, unchecking any single row should automatically downgrade the state back to page-local selection (or clear all) to prevent ambiguous states.
