| Boundary | Description |
|----------|-------------|
| Workflow-directed selection params -> Lifeline row resolution | URL-driven handoff must not let clients invent legal actions or bypass backend authorization. |
| Shared preview envelope -> workflow-native semantics | Any naming generalization must preserve the existing lifecycle and token-consume guarantees. |
| Workflow-native copy -> audit/support truth | Lifeline must not overstate repair or cancel guarantees when rendering workflow actions. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-22-05 | T | shared preview lifecycle | mitigate | Preserve the existing `ready`/`drifted`/`expired`/`consumed` reducer and single-use token checks even if helper names become more general. |
| T-22-06 | I | workflow-native preview and audit copy | mitigate | Render workflow-specific labels and cooperative cancel wording while keeping sensitive details in durable preview or audit rows rather than public telemetry. |
</threat_model>
