<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Durable workflow truth -> rendered allowed actions | Only runtime-owned diagnosis and refusal evidence may decide whether an action is legal. |
| Lifeline backend adapter -> workflow command facade | Lifeline may execute workflow actions, but it must not bypass `Workflow`/`Runtime` legality paths. |
| Cancel request evidence -> operator expectations | Workflow request cancel must remain cooperative and auditable rather than implying an immediate hard stop. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | T | workflow action projection | mitigate | Compute executable actions in `Workflow.Runtime` and `Explain`, then have surfaces consume the normalized result without inference. |
| T-22-02 | T | workflow_request_cancel routing | mitigate | Route workflow-level cancel through `Workflow.request_cancel/3` and verify request-versus-outcome evidence remains intact. |
| T-22-03 | R | refusal and command evidence | mitigate | Preserve durable refusal and command-attempt records so action denials and cooperative cancel stories remain attributable. |
</threat_model>
