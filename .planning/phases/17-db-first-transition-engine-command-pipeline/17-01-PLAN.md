<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public `Workflow.*` wrappers -> internal legality engine | Widening the public API or leaking low-level transaction details would create semver burden and caller confusion. |
| Command rejection -> durable support truth | Returning an error without persisting the refusal would leave operators and maintainers unable to diagnose why a mutation was denied. |
| Runtime schema -> installer and fixture migrations | If schema changes land in code only, supported hosts and repo fixtures will drift from the legal transition contract. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | T | workflow mutation legality | mitigate | Route all accepted verbs through one internal intake path before any row mutation occurs. |
| T-17-02 | R | rejected commands | mitigate | Persist durable mutation-attempt evidence with bounded reason codes rather than audit-only traces. |
| T-17-03 | I | legacy semantics rows | mitigate | Explicitly gate `semantics_version < 2` commands instead of silently upgrading historical meaning. |
| T-17-04 | D | installer / fixture drift | mitigate | Update install and fixture migrations in the same slice as runtime schema changes and verify with grep plus tests. |
</threat_model>
