## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, badge padding trim, dense evidence metadata |
| sm | 8px | Compact stack spacing inside cards and table cells |
| md | 16px | Default paragraph spacing, card padding increments, step row internals |
| lg | 24px | Section padding, panel gutters, top-level content stacks |
| xl | 32px | Two-column panel separation and diagnosis-to-detail spacing |
| 2xl | 48px | Major sub-surface separation below page intro and above raw facts |
| 3xl | 64px | Page-level rhythm when stacking full-width sections on smaller screens |

Exceptions: none. Keep current `p-6`, `gap-6`, `p-4`, and `space-y-6/8` rhythm already established in native operator pages.

Interaction density rules:
- Workflow list rows use `px-4 py-3`.
- Summary/evidence/detail cards use `p-4`.
- Empty states use `p-6`.
- Same-page step links and toggle controls must keep at least 36px vertical hit area even in desktop table layouts.

## Typography

| Body | 14px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 16px | 600 | 1.3 |

Type contract:
- Body copy stays `text-sm` for explanatory and evidence text.
- Section headings inside cards stay `text-base font-semibold`.
- Page title stays `text-2xl font-semibold`.
- Diagnosis labels, evidence labels, and badges use semibold only when naming a durable fact; avoid adding a third weight.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | Primary page surfaces, tables, cards, detail panes |
| Secondary (30%) | `#f8fafc` | Table headers, evidence wells, secondary panel backgrounds |
| Accent (10%) | `#4f46e5` | Inspect/open CTA, selected row state, active step state, active step patch links, diagnosis emphasis for the primary explanation block |
| Destructive | `#dc2626` | Destructive actions only; Phase 21 uses this only for refusal/error framing if needed, not for a new mutation control |

Accent reserved for: `Inspect Workflow` CTA, selected workflow/step highlight, active segmented toggle or patch target, and the single primary diagnosis summary affordance. Do not spend accent on every link, every badge, or generic metadata.

Semantic support colors:
- Warning/caution: use existing amber contract `#fef3c7` / `#92400e` framing for read-only banners, refusal summaries, stale or cautionary evidence, and unsupported guidance.
- Neutral text: keep zinc/slate grayscale for secondary facts and raw internals.
- Success/resolved states may reuse existing emerald treatment from Lifeline only when a diagnosis or terminal fact is positively resolved.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Inspect Workflow |
| Empty state heading | No workflows yet |
| Empty state body | Persist a workflow definition to inspect diagnosis, evidence, and DAG state here. |
| Error state | Workflow details could not be loaded from durable state. Refresh the page or reopen the workflow from the list. If it still fails, verify the workflow row still exists. |
| Destructive confirmation | none in Phase 21; workflow actions remain guidance-only until Phase 22 |

Additional copy rules:
- Read-only banner must stay explicit: operators can diagnose here, but preview, reason, and audited mutations belong to Powertools-native mutation surfaces, not this screen.
- Allowed next action copy must be phrased as legal guidance, not as an executable promise. Use wording equivalent to “Legal next action” or “Allowed next action”, never button-like imperatives.
- When final truth is known, headline copy must favor the specific terminal outcome over stale request evidence. Example contract: show `completed_after_cancel_request` instead of generic `cancel_requested`.
- Unsupported or unknown states must say the surface is showing durable evidence only and that no additional action is available in this phase.

## Surface Contract

Phase 21 interaction model:
- Keep the page read-only.
- Preserve two-column desktop layout: workflow list left, workflow diagnosis summary/context right.
- Preserve single-column mobile flow by stacking list, summary, selected-step detail, then raw facts.
- Default workflow detail landing state is diagnosis summary first, evidence second, full internals third.
- Default selected step is server-chosen from durable evidence priority, not first visible row; URL `?step=` still overrides the default.
- Step changes use LiveView patch navigation so the page does not remount and the selected-step URL remains shareable.

Workflow summary block requirements:
- Show workflow name, durable diagnosis headline, current workflow state, semantics profile, and one allowed-next-action guidance line.
- Show callback posture and latest recovery session only as secondary evidence, below the headline block.
- Show latest refusal in amber framing with legal next steps if present.

Step list requirements:
- Keep the whole DAG visible as context.
- Each step row shows step name, state, step diagnosis, and concise blocker or refusal evidence when present.
- Use accent highlight for the selected step and amber highlight for non-selected blocked steps.
- Do not collapse the DAG into only the primary step; the rest of the graph remains visible around the primary diagnosis.

Selected-step detail requirements:
- The first panel answers what this step is waiting on or what durable result is already known.
- The second panel shows dependency reasons and dependency snapshot facts.
- The third panel exposes raw or policy-filtered result payload and the deep link to generic Oban Web job inspection when `job_id` exists.

Evidence ordering:
1. Diagnosis summary
2. Allowed next action / legal next step guidance
3. Refusal, recovery, callback, and blocker evidence
4. Raw facts and dependency internals
