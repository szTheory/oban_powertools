<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test success -> proof artifact publication | Green tests alone are insufficient if claim evidence files are absent or malformed. |
| CI logs -> published artifacts | Raw logs can contain unsafe content if redaction gates are not enforced. |
| aggregate job result -> merge safety | Aggregate status must fail on missing artifacts and unsafe evidence, not only failed tests. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-39-02-01 | Tampering | Claim matrix files | high | mitigate | Generate per-claim machine-readable files with fixed schema and aggregate from those files only. |
| T-39-02-02 | Information Disclosure | Published logs/artifacts | high | mitigate | Add explicit redaction scan and fail on unsafe patterns before aggregate success. |
| T-39-02-03 | Repudiation | CI evidence integrity | high | mitigate | Include run id, commit SHA, command, and status in metadata files for every claim. |
| T-39-02-04 | Denial of Service | Missing artifact paths | high | mitigate | Use `if-no-files-found: error` and aggregate fail-on-missing boundaries. |
</threat_model>
