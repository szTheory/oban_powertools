## Visuals

### Layout Strategy
The overall `/ops/jobs` operator area will utilize a 3-layer Web UI architecture:
1.  **Top Header:** Global environment/node indicators and user profile/auth status (Sigra integration).
2.  **Sidebar Navigation:** Distinct links separating Powertools Native Shell pages (Limiters, Workflows, Batches, Doctor) from the Oban Web bridge links.
3.  **Main Content Area:** The designated rendering space for either native LiveView pages or the embedded Oban Web viewer.

### Visual Hierarchy
1.  **Global Navigation & Status (High):** Sidebar and top header must persistently anchor the user's context.
2.  **Actionable Controls (Medium):** Bulk actions, filters, and repair triggers within the main content area.
3.  **Data Grids/Lists (Base):** Job rows, limiter states, and workflow steps forming the core content.

### Focal Points
-   "Why is my job blocked?" explainability text.
-   Dry-run repair action previews.
-   Saturated limiter warnings.
