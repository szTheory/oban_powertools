## Milestone Summary

**Decimal Phases:**

- Phase 52.1: Close gap REL-04 — fix verify-published CI (Igniter committed-modules conflict) — inserted after Phase 52 to resolve the REL-04 blocker found during milestone audit.

**Key Decisions:**

- Hex publication at `0.5.0` (not 1.0) — pre-API-freeze: ship to real adopters before freezing the public surface. Bootstrap at `0.0.0` with `Release-As: 0.5.0` footer commit to resolve first release correctly.
- `include-component-in-tag: false` — future tags are `v*` only (no `oban_powertools-v*` duplication after the initial pair).
- RELEASE_PLEASE_TOKEN = PAT (not GitHub App token) — required for workflow triggers. HEX_API_KEY in secrets. Provenance attestation skipped (not available for third-party registries).
- `telemetry_metrics`/`telemetry_poller` optional, gated like `oban_web` — no runtime cost when absent. `Code.ensure_loaded?` guard for actionable error.
- Inserted Phase 52.1 after audit surfaced REL-04 blocker — accepted the insertion rather than deferring to v1.7.

**Issues Resolved:**

- REL-04 Igniter committed-modules conflict: Phase 52.1 rm-step fix.
- Phase 47 release-please `Release-As: 0.5.0` footer commit pitfall (bootstrap sha must be `0.0.0`, not `0.1.0`).
- `MIX_ENV` inconsistency in `verify-published` compile step.
- Hardcoded `~> 0.5` sed pattern replaced with version-agnostic regex.

**Issues Deferred:**

- Live CI E2E run for REL-04 — requires real release cycle (Phase 52.1 fix is in place; passes on next release).
- Phase 47 missing `VERIFICATION.md` (process gap; deliverable evidence is solid: 0.5.0 live, hexdocs renders, CHANGELOG populated).
- `mix.exs` `igniter` dep scope: intentionally un-scoped (requires all envs for install task); `47-02-SUMMARY.md` documents the stale `:only: [:dev, :test]` note.
