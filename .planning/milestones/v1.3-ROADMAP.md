# Milestone v1.3: Unified Control Plane & Explainability

**Status:** ✅ SHIPPED 2026-05-26
**Phases:** 27-31
**Total Plans:** 15

## Overview

Oban Powertools v1.3 turns the native `/ops/jobs` shell into one coherent operator control plane. The milestone freezes shared control-plane vocabulary, makes the overview diagnosis-first, preserves context across native and bridge handoffs, unifies bounded mutation posture, and closes the public docs and proof chain around an honest native-shell versus bridge-only contract.

## Milestone Summary

**Key Decisions:**

- Freeze one shared operator vocabulary and ownership model across overview, cron, limiters, workflows, Lifeline, audit, and Oban Web handoffs.
- Make `/ops/jobs` the diagnosis-first operator entrypoint rather than a count-heavy landing page.
- Keep mutation scope bounded and native while making preview, reason, refusal, and audit posture read as one policy story.
- Preserve bridge honesty: `/ops/jobs/oban` remains inspection-only rather than a stealth native queue or job rewrite.
- Close support truth through docs-contract, LiveView proof, and example-host lanes instead of adding a broad browser-E2E harness.

**Issues Resolved:**

- Eliminated control-plane wording drift across native pages, docs, and host proof lanes.
- Restored context-preserving native and bridge handoffs so drill-down diagnosis survives refresh, remount, and read-only access.
- Unified audit identity and follow-up linking so audit evidence reads as part of the same operator story.

**Issues Deferred:**

- `QRY-01`: native queue and generic job inspection parity with the bounded bridge.
- `SRE-01`: richer historical forensics, evidence bundles, and runbook-guided remediation.
- `API-02`: CLI or API control-plane diagnosis and bounded-action contracts for automation.

**Technical Debt Incurred:**

- The milestone closed from a dirty working tree, so the release archive captures the shipped state and proof chain without fully atomic per-plan code commits.
