## Igniter conflict fix

**Context:** `oban_powertools_auth.ex` and `oban_powertools_display_policy.ex` are committed in `examples/hex_consumer/lib/hex_consumer_web/`. When `mix oban_powertools.install` runs in CI via `Igniter.Project.Module.create_module`, it finds both files already exist, emits `:error issues`, skips migration generation, and exits 0. Subsequent steps fail.

### Fix approach

| Option | Description | Selected |
|--------|-------------|----------|
| rm in CI before install | Add a pre-install step in verify-published job to rm the two files; committed scaffold unchanged for local dev | ✓ |
| Remove files from scaffold | Un-commit auth/display_policy; Igniter generates them on every install; breaks local dev without running install first | |
| Gitignore the two files | Keep committed; gitignore so CI doesn't check them out; same net effect as rm but more permanent | |

**User's choice:** rm in CI before install
**Notes:** Keeps local dev working without any change to the committed scaffold.

### Fix scope

| Option | Description | Selected |
|--------|-------------|----------|
| Just the rm step | Minimum correct fix | |
| rm step + update regenerate.sh docs | Also add comment explaining that auth/display_policy are committed as local dev scaffolding but overwritten on fresh install | ✓ |

**User's choice:** rm step + update regenerate.sh docs
