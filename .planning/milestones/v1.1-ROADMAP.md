# Milestone v1.1: Host Contract & Adoption Hardening

**Status:** ✅ SHIPPED 2026-05-23
**Phases:** 8-15
**Total Plans:** 27

## Overview

Oban Powertools v1.1 hardens the public host contract around installation, router and supervision ownership, auth and display-policy seams, native-first operator UX, optional `oban_web` support, audited first-session proof, and a real supported upgrade lane. The milestone closes the remaining adoption-trust gaps after v1 by making the documented host path executable, support-truth explicit, and the proof stack honest across fresh-host, native-only, bridge, first-session, and upgrade scenarios.

## Milestone Summary

**Key Accomplishments:**

- Froze the host-owned install, router, supervision, auth, display-policy, and telemetry contract around `/ops/jobs`.
- Repaired the fresh-host installer and canonical fixture so day-0 install, reset/migrate, boot, and first-session proof all run end to end.
- Proved native-only operation without `oban_web` while preserving a bounded read-only `/ops/jobs/oban` bridge lane.
- Repaired the Phase 8-10 evidence chain so `POL-01`, `POL-02`, `POL-03`, and `HST-02` are audit-closeable again.
- Replaced the synthetic upgrade path with a real archived-host upgrade lane.
- Unified README, guides, hardening, troubleshooting, and fixture docs around one explicit support-truth vocabulary.

**Key Decisions:**

- Keep the host app fully responsible for router scope, auth, display policy, supervision wiring, and operator seed state.
- Treat the native `/ops/jobs` shell as the primary operator surface and keep the `oban_web` bridge explicitly narrower and read-only.
- Preserve canonical proof ownership in phase-local verification artifacts while using later repair phases only to restore traceability.

**Issues Resolved:**

- The broken fresh-host installer path and incomplete fixture migrations were closed in Phase 12.
- Native-only support and optional dependency proof gaps were closed in Phase 13.
- The Phase 8-10 requirement evidence chain was repaired in Phase 14.
- The synthetic upgrade lane and overstated support-truth docs were replaced with a real archived-host lane and aligned public guidance in Phase 15.

**Issues Deferred:**

- None at milestone close.

**Technical Debt Incurred:**

- The next milestone still needs fresh requirements and roadmap definition before implementation resumes.
