### Integration Gaps

**INT-01: Doctor manifest missing `oban_powertools_job_records`**
- File: `lib/oban_powertools/doctor/checks.ex`, lines 26–59
- Affected REQ: REC-02
- Impact: Doctor reports all Powertools tables present even if Phase 55 install migration was never run. Hosts that skip the migration get silent `{:error, :not_found}` from `fetch_result/1` with no health signal.
- Fix: Add `oban_powertools_job_records` to a migration set in `@powertools_manifest` (new "output-recording" group).

**INT-02: Cron path does not inject `__deadline_at__` meta**
- File: `lib/oban_powertools/cron.ex`, line 447
- Affected REQs: SAFE-02, SAFE-03 (transitively for cron-scheduled workers)
- Impact: Workers with `deadline:` configured that are also used in cron schedules will have no `__deadline_at__` in cron-inserted job meta. `Deadlines.expired?` in `__powertools_perform__` silently skips deadline cancellation for those jobs.
- Fix: In `maybe_insert_job`, after resolving the worker module, call `Deadlines.build_meta(worker_module.__powertools_deadline_ms__(), DateTime.utc_now())` and merge into opts before `worker_module.new/2`, matching the idempotency path pattern. Or have `new/2` itself inject deadline meta when `function_exported?(worker_mod, :__powertools_deadline_ms__, 0)`.
- Note: Redaction fires correctly on the cron path via the `new/2` override — this gap is specific to deadline meta.
