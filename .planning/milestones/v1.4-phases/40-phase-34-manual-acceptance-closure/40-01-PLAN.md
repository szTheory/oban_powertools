## Automated-Only Gate — No Human UAT

All Phase 34 acceptance evidence is produced by deterministic tests under `mix test --seed 0`. There is no human reviewer step.

### Proxy 1 — Visual hierarchy (stands in for `Overview visual scan`)

Location: `test/oban_powertools/web/live/engine_overview_live_test.exs` (new test inside the existing module).

Assertions:
- Bucket-grid headings appear in DOM order before any historical exemplar or `attention_reason` block: `Diagnosis-first overview` → `Needs Review` → `Blocked` → `Waiting` → `Runnable` → `Bridge-only Follow-up` → `Resolved Recently`.
- The historical exemplar markers (`attention_reason`, `Open forensic timeline`, `Open runbook entry`) render *after* all bucket headings, never as a standalone top-level region.
- No top-level standalone feed-like section header (`Event Feed`, `Activity Feed`, `Event Stream`, `Recent Activity`) is present anywhere in the rendered HTML.

### Proxy 2 — Copy contract (stands in for `Runbook copy judgment`)

Location: `test/oban_powertools/web/live/runbook_copy_contract_test.exs` (new file).

Scope: render `ForensicsLive` for at least one workflow bundle and one Lifeline bundle, and assert the combined HTML across both views satisfies the contract below.

Required markers (must be present on runbook surfaces):
- Ownership triad strings: `Powertools-native`, `Oban Web bridge`, `host-owned follow-up`.
- Evidence-boundary labels: at least one of `partial evidence`, `history unavailable`, `unknown`.
- Refusal/runbook section ordering across the rendered surfaces: `Outcome` → `Reason` → `Legal next move` → `Venue`.

Forbidden markers (must be absent from runbook surfaces):
- Execution over-claims: `executed remediation`, `completed remediation`, `delivered alert`, `alert delivery`, `runbook session`, `session persists`, `persisted session`, `we will execute`, `we executed`.
- Faux-native action shortcuts on bridge/host-owned paths: `phx-click=".*runbook"`, `checklist`.

## Evidence Contract

- CI evidence: green runs of the two proxy suites under `--seed 0`, exercised by Phase 40 Plan 02's C3/C4 lane extensions.
- UAT log: `reviewer: automated`, `result: pass`, test paths and assertion summaries recorded.

No requirement status promotion is allowed without both the proxy tests being green and being referenced from `34-UAT.md`.
</rubric>
