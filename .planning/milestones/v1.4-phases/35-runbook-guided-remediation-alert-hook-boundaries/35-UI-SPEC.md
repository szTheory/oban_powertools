# Phase 35 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Phoenix LiveView HEEx with existing utility-class patterns |
| Icon library | none |
| Font | host/system sans-serif inherited by `/ops/jobs` |

Source: the codebase has no `components.json`, Tailwind config, PostCSS config, React/TSX tree, or frontend CSS token pipeline. Phase 35 stays on Phoenix LiveView surfaces, so shadcn initialization is not applicable.

Primary focal point: the runbook continuity panel is the first visual anchor on remediation-capable views. It must show, in order, diagnosis summary, selected legal next path, venue/ownership, and current attempt status before secondary metadata or follow-up links.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline metadata separators, compact status badge padding |
| sm | 8px | Label-to-value spacing, small stacked evidence rows |
| md | 16px | Default card/table cell padding and section internals |
| lg | 24px | Separation between runbook continuity panel and evidence/audit blocks |
| xl | 32px | Separation between page intro, state summary, and detail content |
| 2xl | 48px | Major section breaks for deep forensic layouts only |
| 3xl | 64px | Reserved for page-level spacing only; avoid inside cards |

Exceptions: keep existing action controls at minimum 32px height; icon-only controls require a minimum 48px touch target and text fallback. Host-owned follow-up guidance rows may use compact `px-2 py-1` labels when paired with explicit ownership text.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 600 | 1.4 |
| Heading | 16px | 600 | 1.3 |

Only two weights are allowed: regular `400` and semibold `600`. Attempt status, ownership labels, and evidence completeness must stay at Label or Body size; no additional headline tier is allowed in this phase.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Primary content panels, remediation continuity card backgrounds, table/list bodies |
| Secondary (30%) | `#F8FAFC` | Header strips, nested evidence groups, host-follow-up guidance containers |
| Accent (10%) | `#4F46E5` | Primary native CTA, active row/selection state, forensic/audit continuity links |
| Destructive | `#DC2626` | Destructive confirmations and failed native execution states only |

Accent reserved for: `Preview Native Remediation`, selected diagnosis row emphasis, `Open forensic evidence`, and `Open audit record` links. Accent must not style bridge-only or host-owned rows as native controls.

Supporting semantic tones already in use remain valid: amber for caution/refusal/partial evidence, slate for unknown or read-only states, emerald for confirmed successful native completion only.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Preview Native Remediation |
| Empty state heading | No remediation attempts recorded yet |
| Empty state body | This diagnosis has not entered a supported native remediation flow. Review legal next paths, then start a native preview to capture attempt context. |
| Error state | Runbook continuity data is unavailable for this attempt. Refresh the view, reopen preview, and retry from the supported native path. |
| Destructive confirmation | Execute Remediation: This writes a native remediation attempt to audit and forensic evidence. Confirm only after reviewing reason, ownership, and expected outcome. |

Additional copy rules:
- Preserve ownership triad wording exactly: `Powertools-native`, `Oban Web bridge`, `host-owned follow-up`.
- Keep reading order consistent: `diagnosis -> runbook guidance -> legal next path -> venue/ownership -> evidence -> audit follow-up`.
- Host fallback states must be explicit and honest, for example `Host-owned follow-up unavailable` or `No host escalation hook configured`.
- Never imply first-party alert delivery, paging, or ticket creation by Powertools core.

---

## Phase Interaction Contract

Phase 35 extends existing surfaces by making remediation continuity visible and durable. It does not add a new incident console, paging center, or provider destination manager.

Whenever a supported native path is previewed or executed, the UI must render:
- Which runbook entry or legal next path led here.
- Why the action was allowed (diagnosis state, evidence completeness, ownership/venue).
- What attempt state is recorded (`previewed`, `attempted`, `succeeded`, `refused`, `drifted`, `expired`, `consumed`, `bridge-only`, `host-owned follow-up`).
- Where to inspect durable evidence and audit follow-up.

Bridge-only and host-owned follow-up paths remain guidance/link treatments, never filled primary native controls.

## Surface Contracts

| Surface | Required Phase 35 Behavior |
|---------|----------------------------|
| `/ops/jobs/lifeline` | Show runbook continuity details adjacent to preview/execute actions; preserve preview-first flow and explicit reason capture before execution. |
| `/ops/jobs/forensics` | Render canonical attempt continuity from durable evidence: source runbook path, attempt state, ownership/venue, and audit linkage. |
| `/ops/jobs/workflows` | Keep mutation venue truthful; workflow surface can route to Lifeline/native or evidence follow-up but must not become a second mutation venue. |
| `/ops/jobs/cron` and `/ops/jobs/limiters` | Preserve diagnosis-first guidance; host-owned and bridge-only follow-up remain visually distinct from native action controls. |
| Shared presenter copy | Continue central wording for ownership and fallback boundaries; no per-page drift in escalation truth language. |

## State Contract

| State | Visual Treatment | Required Copy Behavior |
|-------|------------------|------------------------|
| Previewed | Secondary panel with action-intent badge | Explain that a native action is prepared but not yet executed; include next legal action. |
| Attempted | Dominant panel + continuity metadata strip | State action, target, and reason context with durable evidence link. |
| Succeeded | Dominant panel with success accent only in status element | Confirm completion fact and point to audit record/evidence bundle. |
| Refused / Drifted / Expired / Consumed | Amber boundary panel with explicit state label | Explain why execution did not proceed and what legal path remains. |
| Bridge-only | Bordered guidance row, no filled primary button | Use `Oban Web bridge` plus `inspection only` posture. |
| Host-owned follow-up | Bordered or muted guidance row, no native action affordance | Use `host-owned follow-up` and explicit configured/unconfigured truth text. |
| Unknown / partial evidence | Secondary panel with boundary note | Use `partial evidence` or `unknown` explicitly; avoid certainty claims. |
