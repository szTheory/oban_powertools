# Requirements Archive: Oban Powertools v1.7 Worker Lifecycle & Safety

**Archived:** 2026-06-13  
**Status:** ✅ All 18 requirements COMPLETE

---

## Overview

Equip every `ObanPowertools.Worker` with observable, durable lifecycle hooks, a generalised output recording contract, and at-rest redaction before adding Batches.

**Milestone:** v1.7  
**Phase range:** 53-56

---

## v1.7 Requirements (All Complete)

### Worker Lifecycle Hooks

- [x] **HOOK-01**: Worker can declare `on_start/1` callback that fires before `process/1` — observe-only, crash-caught, no-op default ✓ v1.7 Phase 53
- [x] **HOOK-02**: Worker can declare `on_success/2` callback that fires when `process/1` returns `:ok` or `{:ok, _}` — observe-only, crash-caught, no-op default ✓ v1.7 Phase 53
- [x] **HOOK-03**: Worker can declare `on_failure/2` callback that fires when `process/1` returns `{:error, _}` or raises — observe-only, crash-caught, no-op default ✓ v1.7 Phase 53
- [x] **HOOK-04**: Worker can declare `on_discard/2` callback that fires when job is discarded after exhausting all retries — observe-only, crash-caught, no-op default ✓ v1.7 Phase 53
- [x] **HOOK-05**: Worker hook invocations emit telemetry under a new `worker_hook` family in the frozen low-cardinality contract (`hook`, `outcome` keys) ✓ v1.7 Phase 53

### Deadline & Timeout

- [x] **SAFE-01**: Worker can declare `timeout: milliseconds` in `use ObanPowertools.Worker` opts to generate a compile-time `timeout/1` callback default ✓ v1.7 Phase 54
- [x] **SAFE-02**: Worker can declare `deadline: duration` in `use ObanPowertools.Worker` opts; job stores `__deadline_at__` ISO8601 timestamp in meta at enqueue time ✓ v1.7 Phase 54 (idempotency path; cron path gap — INT-02 deferred to v1.8)
- [x] **SAFE-03**: `perform/1` checks `__deadline_at__` before calling `process/1`; returns `{:cancel, :deadline_expired}` if wall-clock deadline has passed ✓ v1.7 Phase 54
- [x] **SAFE-04**: `mix oban_powertools.doctor` surfaces `retryable` jobs whose `__deadline_at__` has passed as a warning ✓ v1.7 Phase 54

### Output Recording

- [x] **REC-01**: Worker can declare `record_output: true` in `use ObanPowertools.Worker` opts to opt in to persisting `{:ok, payload}` return values from `process/1` ✓ v1.7 Phase 55
- [x] **REC-02**: `ObanPowertools.JobRecord` Ecto schema with `oban_powertools_job_records` table — standalone-job output storage independent of `Workflow.Result` ✓ v1.7 Phase 55 (Doctor manifest gap — INT-01 deferred to v1.8)
- [x] **REC-03**: Host can retrieve latest recorded output for a job via `fetch_result/1` returning `{:ok, result}` or `{:error, :not_found}` ✓ v1.7 Phase 55
- [x] **REC-04**: Recorded output is visible in the `/ops/jobs` job detail view via a new `:job_recorded` DisplayPolicy kind ✓ v1.7 Phase 55
- [x] **REC-05**: Worker can declare `output_limit: bytes` (byte cap) and `output_retention: policy` (`:standard`/`:extended`/`:ephemeral`) as compile-time opts ✓ v1.7 Phase 55

### At-rest Redaction

- [x] **REDACT-01**: Worker can declare `redact: [:field]` in `use ObanPowertools.Worker` opts; listed fields are dropped from `args` via `Map.drop` at enqueue time, strictly after idempotency fingerprint calculation ✓ v1.7 Phase 56
- [x] **REDACT-02**: Redacted field names stored in job meta as `__redacted_fields__` at enqueue time ✓ v1.7 Phase 56
- [x] **REDACT-03**: `/ops/jobs` job detail view renders "Fields redacted at enqueue: [:field]" from `meta["__redacted_fields__"]` ✓ v1.7 Phase 56
- [x] **REDACT-04**: `DisplayPolicy.render_job_field/3` default rendering shows "Redacted at enqueue" for fields listed in `__redacted_fields__` meta ✓ v1.7 Phase 56

## Future Requirements (Deferred from v1.7)

*(Revisit in v1.8 or later)*

- INT-01 fix: Add `oban_powertools_job_records` to `@powertools_manifest` in Doctor — deferred to v1.8
- INT-02 fix: Inject `__deadline_at__` meta on cron path in `maybe_insert_job` — deferred to v1.8
- Global `attach_hook/1` registry across workers — defer until adoption signal
- Hard deadline (interrupt running job) — requires Oban Pro DynamicLifeline supervision pattern
- `encrypt:` at-rest — defer indefinitely (collides with fingerprint, blinds job filter, leaks via meta/errors)
- Retroactive redaction (`Redactor.scrub_past_jobs/2`) — requires full Lifeline preview/reason/audit wrapping
- `on_discard` from Lifeline discard path (operator-initiated discards) — v1.7 scope covers Oban-native retry exhaustion only
- `record_output/2` callable explicitly from inside `process/1` without auto-recording — evaluate after v1.7 adopter feedback
