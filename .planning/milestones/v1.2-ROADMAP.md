# Milestone v1.2: Workflow Semantics & Recovery

**Status:** ✅ SHIPPED 2026-05-25
**Phases:** 16-26
**Total Plans:** 31

## Overview

Oban Powertools v1.2 turns workflows from durable DAG primitives into a support-truthful orchestration substrate. The milestone freezes lifecycle vocabulary, DB-first mutation legality, durable callback and recovery evidence, await/signal/expiry authority, diagnosis-first workflow surfaces, bounded Lifeline actions, and the proof posture needed to explain in-flight workflow state under real host-app conditions.

## Milestone Summary

**Key Decisions:**

- Treat semantics version `2` as the explicit durable workflow lifecycle contract.
- Keep workflow truth DB-first so runtime and operator actions re-enter one legal command path.
- Preserve callbacks, recovery attempts, awaits, signals, and cancel requests as durable evidence instead of implicit transient behavior.
- Keep the workflow page diagnosis-first and route bounded execution through Lifeline.
- Separate the singular supported host upgrade lane from broader repo-local compatibility proof.

**Issues Resolved:**

- Closed the missing `VERIFICATION.md` gaps for workflow-semantic phases 17 and 19-23.
- Repaired v1.2 requirement traceability so owner phases, proof files, and audit results match.
- Normalized the historical Phase 12 UAT artifact and cleared the remaining archival blocker.

**Issues Deferred:**

- `REC-04`: preview exact workflow recovery mutations across every path.
- `CBK-01`: broaden callback policy beyond terminal and recovery hooks.
- `CTL-01`: unify cron, limiters, workflows, queues, and Lifeline under one control-plane vocabulary.
- `API-01`: expose workflow control-plane actions over CLI or API contracts.

**Technical Debt Incurred:**

- Several late-phase workflow milestones were closed from an already-dirty working tree, so verification artifacts carry canonical proof but not full per-task atomic commit history.
