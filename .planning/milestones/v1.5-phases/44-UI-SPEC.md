## Component Inventory

### Job Detail Page (`/ops/jobs/jobs/:id`) - Action Header Additions

**Focal Point**: The Action Bar is the primary focal point of the header area, drawing the operator's eye to available mutation actions (Retry, Cancel, Discard). In the modal, the Confirmation Button is the focal point.

1. **Action Bar** — Located in the page header next to the H1, or top-right of the Identity card:
   - `div.flex.gap-2.items-center`
   - **Retry Button** — `button.rounded.bg-white.px-4.py-2.text-sm.font-semibold.text-indigo-600.border.border-indigo-200.hover:bg-indigo-50` (appears for retryable, discarded, cancelled, completed states)
   - **Cancel Button** — `button.rounded.bg-white.px-4.py-2.text-sm.font-semibold.text-red-600.border.border-red-200.hover:bg-red-50` (appears for available, scheduled, executing, retryable states)
   - **Discard Button** — `button.rounded.bg-white.px-4.py-2.text-sm.font-semibold.text-red-600.border.border-red-200.hover:bg-red-50` (appears for available, scheduled, executing, retryable states)

### Action Preview Modal (Lifeline Pipeline)

1. **Modal Overlay** — `div.fixed.inset-0.bg-zinc-900/50.backdrop-blur-sm` (standard Phoenix CoreComponents modal)
2. **Modal Surface** — `div.relative.bg-white.rounded-lg.shadow-xl.p-6`
3. **Modal Title** — `h2.text-base.font-semibold` ("Retry Job", "Cancel Job", or "Discard Job")
4. **Preview Panel** — `div.mt-4.rounded.bg-slate-50.p-4.text-sm`
   - Displays the job ID, current state, and the action to be taken.
5. **Reason Input** — `form.mt-4.space-y-4`
   - `label.block.text-sm.font-semibold` "Reason"
   - `input[type="text"].w-full.rounded-md.border-gray-300.text-sm` (required, cannot be empty)
6. **Concurrent Guard Alert** — `div.mt-4.rounded.bg-red-50.p-4.text-sm.text-red-800.border.border-red-200`
   - Appears ONLY if the mutation fails due to state change (e.g. "Job state has changed since you opened this page.")
7. **Action Footer** — `div.mt-6.flex.justify-end.gap-4`
   - **Cancel/Close** — `button.text-sm.font-semibold.text-slate-600`
   - **Confirm Action** — `button.rounded.bg-indigo-600` (for Retry) or `bg-red-600` (for Cancel/Discard) `.px-4.py-2.text-sm.font-semibold.text-white`
   - Confirm button must be disabled until reason string is entered.

## Interaction Contract

### Mutation Flow (Preview -> Reason -> Execute)
1. User clicks Retry, Cancel, or Discard on the job detail page.
2. Modal opens presenting a preview of the action.
3. User must type a reason string in the text input. The "Confirm" button is `disabled` until `String.trim(reason) != ""`.
4. User clicks "Confirm". The form submits via `phx-submit`.
5. LiveView calls `Lifeline.execute_repair/4`. No direct Oban calls.
6. On success: Modal closes, success flash is shown, detail page state is re-fetched and updated.
7. On concurrent modification error (another operator changed state): Modal stays open (or error is shown inline), visible error alert is presented: "The job's state was changed by another process or operator."
8. `"job_discard"` is an explicitly supported Lifeline action, handled just like `"job_cancel"` and `"job_retry"`.

### Unauthorized / Read-Only State
- If `@read_only?` is true (via `LiveAuth.authorize_page/3`), the mutation buttons (Retry, Cancel, Discard) are disabled or completely omitted. The read-only banner defined in Phase 43 remains visible on the job detail page.
