<interfaces>
Existing read-model analog:
- `ObanPowertools.Jobs` defines a filter struct in the module itself, accepts `repo` as the first argument, imports `Ecto.Query`, and is read-only.

Required `ObanPowertools.Batches` contract:
- `%Batches{status: :all, query: nil, chain_only: false, queue: nil, worker: nil, page: 1, page_size: 20}` is the filter struct.
- `list(repo, %Batches{} = filter, opts \\ [])` returns batch list rows.
- `get(repo, batch_id, opts \\ [])` returns `nil` or a detail struct/map with batch identity, failed members, callbacks, chain context, blocked state, and audit evidence.
- `count_by_status(repo, %Batches{} = base_filter)` returns keys `all`, `inserting`, `executing`, `exhausted`, `insert_failed`, `callback_failed`, and `completed`.
- `blocked_state(detail_or_batch, now \\ DateTime.utc_now())` returns copy-ready state data for `insert_failed`, `callback_failed`, `output_unavailable`, `output_expired`, `executing`, `exhausted`, or `completed`.
</interfaces>
</context>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| read model filters to database | Filter values influence Ecto queries and must remain parameterized. |
| persisted payloads to operator display | Job args/meta, callback payloads, and error-like values may contain sensitive data. |
| retry eligibility to UI controls | Eligibility must be server-derived, not inferred from client-controlled events. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-62-01 | Tampering | `Batches.list/3` filters | mitigate | Use Ecto query parameters and closed status values; tests cover filter behavior. |
| T-62-02 | Information Disclosure | detail payload rendering | mitigate | Use `DisplayPolicy.render_job_field/3` seams for job args/meta/callback payloads and avoid raw `inspect/1` rendering. |
| T-62-04 | Tampering | callback retry eligibility | mitigate | Derive callback `retry_eligible?` from durable status and lease expiry inside the read model. |
| T-62-07 | Repudiation | bulk retry outcomes | mitigate | Expose retryable member ids and directly findable audit evidence so later UI reports successes/skips/failures honestly. |
| T-62-SC | Tampering | package installs | accept | No package-manager install tasks exist. |
</threat_model>
