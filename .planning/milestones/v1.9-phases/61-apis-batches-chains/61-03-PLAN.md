<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer chain DSL to persisted job metadata | Chain specs become batch rows and Oban job metadata consumed by chain callback dispatch. |
| Developer dynamic args to durable builder refs | Builder references may be persisted and executed by Powertools callback progression. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-61-01 | Denial of Service | chain step count and metadata | mitigate | Validate nonempty linear steps and reject branch-like inputs; insert one first job and leave progression to callback dispatch. |
| T-61-02 | Elevation of Privilege | dynamic chain builders | mitigate | Reject anonymous functions and persist only `{Module, :function, extra_args}` references; Plan 61-05 adds builder behavior enforcement before execution. |
| T-61-03 | Information Disclosure | chain metadata | mitigate | Store ids, names, step descriptors, and builder references only; do not copy upstream output payloads into job args or metadata. |
| T-61-04 | Repudiation | output-dependent steps | transfer | This plan records metadata needed for output-dependent steps; Plan 61-05 returns explicit errors for unavailable outputs. |
| T-61-SC | Tampering | Package installs | mitigate | No package-manager install tasks exist in this plan; RESEARCH.md Package Legitimacy Audit lists no packages. |
</threat_model>
