# hex_consumer

A minimal Phoenix application that consumes `oban_powertools` as a **true hex
dependency** — `{:oban_powertools, "~> 0.5"}` resolved via `mix deps.get` from
hex.pm.

## Purpose

This example is the verification target for REL-04: it proves that the
**published hex tarball** works end-to-end for a fresh adopter, exercising the
`:files` whitelist, the embedded migration bodies in the installer, and the
full operator session surface at `/ops/jobs`.

It is **distinct from `examples/phoenix_host`**, which uses a `path: "../.."`
dep and tests in-repo changes. The hex consumer tests what ships to adopters.

## Key differences from examples/phoenix_host

- **Dep source:** `{:oban_powertools, "~> 0.5"}` (hex) vs `path: "../.."` (local).
- **No `oban_web` dep:** This consumer exercises the **native-only** operator
  path. The first-session test explicitly asserts `refute html =~ "Oban Web"`.
  This is intentional — not a bug (D-06).
- **No committed `mix.lock`:** Lock is gitignored so CI always resolves fresh,
  mirroring what an adopter does on first install.
- **No committed migrations:** `priv/repo/migrations/` is gitignored and
  generated by `mix oban_powertools.install` at setup time.

## Requirements

This app **requires hex.pm reachability** to install or run locally. It cannot
be used offline or against locally-modified `oban_powertools` source without
temporarily switching the dep to `path: "../.."`.

## Local setup

```sh
# Requires hex.pm access
mix deps.get
mix oban_powertools.install
mix ecto.setup
mix test test/hex_consumer_web/oban_powertools_first_session_test.exs
```

## regenerate.sh

`regenerate.sh` is the maintainer companion for regenerating this fixture from
a fresh `mix phx.new` baseline. It inserts the hex dep (not a path dep) and
runs `mix oban_powertools.install`. Run only when `oban_powertools` is live on
hex.pm — it requires internet access by design.
